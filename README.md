## 1. 기존 기획사항 정리 (IPm 기준)

### 1.1 IPm DEG 처리 흐름
- DEG(Differentially Expressed Gene) 목록을 입력으로 받아 oligo 디자인 후보 서열을 생성
- 발현 변화량(fold change), 통계적 유의성(p-value / FDR) 기준으로 DEG 필터링
- 필터링된 DEG에 대해 타겟 서열 추출 및 oligo 후보 설계

### 1.2 Oligo 디자인 규칙 (IPm 기준)
| 항목 | IPm 기준값 |
|------|-----------|
| 길이 | 20–25 nt |
| GC 함량 | 40–60% |
| Tm (녹는점) | 55–65 °C |
| 헤어핀 구조 | 허용 안 함 (ΔG > −2 kcal/mol) |
| 오프타겟 결합 | BLAST 기반 단순 유사도 점검 |
| 중복 서열 제거 | 동일 유전자 내 중복 제거 |

### 1.3 DEG 선별 기준 (IPm 기준)
- |log₂ fold change| ≥ 1.5
- Adjusted p-value (FDR) ≤ 0.05
- 발현량(normalized count) ≥ 10 (저발현 유전자 제외)

### 1.4 Known 이슈 및 한계
- 오프타겟 예측이 BLAST 단순 유사도 기반이라 특이성 정밀도 낮음
- Tm 계산 시 nearest-neighbor 모델 미적용 → 실험값과 괴리 발생
- 동일 유전자 내 isoform 간 서열 겹침(overlap) 처리 미흡
- 대규모 DEG 목록(≥ 500개) 처리 시 속도 저하

## 2. Oligo 디자인 규칙 기준 IPm vs IPm2 적용사항 및 차이

### 2.1 규칙 비교표

| 항목 | IPm (기존) | IPm2 (신규) | 변경 사유 |
|------|-----------|------------|---------|
| 서열 길이 | 20–25 nt | 18–30 nt (유전자별 최적화) | 유전자 구조 다양성 반영 |
| GC 함량 | 40–60% | 45–55% (엄격화) | 증폭 효율 및 특이성 개선 |
| Tm 계산 방식 | 단순 공식 (Wallace rule) | Nearest-neighbor 모델 (SantaLucia 1998) | 실험값과의 일치율 향상 |
| 헤어핀 판정 | ΔG > −2 kcal/mol | ΔG > −3 kcal/mol (완화), 구조별 세분화 | 과도한 후보 탈락 방지 |
| 오프타겟 검사 | BLAST 단순 유사도 | BLAST + 열역학 결합 에너지(ΔG) 복합 검사 | 오프타겟 특이성 정밀도 향상 |
| Isoform 처리 | 미지원 | 유전자별 공통 엑손 영역 우선 타겟팅 | isoform 간 coverage 균형 확보 |
| 중복 제거 범위 | 동일 유전자 내 | 유전자군(gene family) 수준 확장 | 유사 서열 유전자 간 교차반응 방지 |
| DEG 우선순위 | fold change 절댓값 순 | fold change + 발현 수준(TPM) 복합 스코어 순 | 생물학적 유의성 높은 DEG 우선 선별 |
| 배치 처리 | 순차 처리 | 병렬 처리 지원 | 대용량 DEG 목록 처리 성능 개선 |

### 2.2 주요 변경 포인트 요약
1. **Tm 계산 정밀화**: Wallace rule → SantaLucia nearest-neighbor 모델 전환으로 Tm 예측 오차 감소
2. **오프타겟 검사 강화**: 서열 유사도 단독 → 열역학 결합 에너지 복합 평가로 위음성(false negative) 감소
3. **Isoform-aware 설계**: 공통 엑손 우선 타겟팅으로 isoform 간 coverage 불균형 해소
4. **성능 개선**: 병렬 처리 도입으로 대규모 DEG 처리 시간 단축

## 3. IPm2 DEG 관련 개선

### 3.1 배경

IPm은 DEG 기반 oligo 설계 자동화를 목표로 개발된 초기 버전 도구이다. 실제 실험 환경에 적용하면서 다음과 같은 문제가 누적되었다:

- RNA-seq 분석 기술의 발전으로 DEG 목록 규모가 급격히 증가(수백 → 수천 개 수준)
- 단순 Tm 계산 및 오프타겟 검사 방식의 한계로 설계된 oligo의 실험 성공률 저조
- Isoform 다양성이 높은 유전자군에서 특정 isoform만 타겟팅되는 편향 발생
- 연구자 요구에 따른 커스텀 디자인 규칙 적용이 어려운 경직된 구조

이러한 문제를 체계적으로 해결하기 위해 IPm2를 기획하였다.

### 3.2 필요성

| 문제 | 영향 | 개선 필요도 |
|------|------|------------|
| Tm 계산 부정확 | 실험 최적화 온도와 차이 발생 → 재설계 비용 증가 | 높음 |
| 오프타겟 검사 정밀도 부족 | 비특이적 결합 위험 → 실험 신뢰성 저하 | 높음 |
| Isoform 처리 미흡 | 특정 isoform 편향 → 유전자 발현 해석 왜곡 | 중간 |
| 대용량 처리 성능 저하 | 처리 시간 증가 → 연구 사이클 지연 | 중간 |
| 규칙 커스터마이징 불가 | 연구 목적별 맞춤 설계 불가 → 범용성 제한 | 중간 |

### 3.3 목적

IPm2 DEG 개선의 주요 목적은 다음과 같다:

1. **실험 성공률 향상**
   - nearest-neighbor 기반 Tm 계산 및 열역학 오프타겟 검사 도입으로 설계 품질 제고
   - 결과적으로 oligo 재설계 빈도 감소 및 실험 비용 절감

2. **Isoform-aware 설계 지원**
   - 공통 엑손 기반 타겟팅 전략 도입으로 유전자 수준의 균형 잡힌 coverage 확보
   - 복잡한 대안 스플라이싱(alternative splicing) 유전자에 대한 분석 신뢰성 향상

3. **대규모 DEG 처리 성능 확보**
   - 병렬 처리 아키텍처 적용으로 수천 개 DEG에 대한 실시간 설계 지원

4. **유연한 규칙 커스터마이징**
   - 연구 목적 및 실험 조건에 따른 디자인 파라미터 조정 기능 제공
   - 내부 기준 외 사용자 정의 필터 규칙 추가 지원

5. **재현성 및 추적성 확보**
   - 설계 이력 및 적용 규칙 버전 관리 기능 도입
   - 결과 리포트 표준화로 연구 재현성 보장
